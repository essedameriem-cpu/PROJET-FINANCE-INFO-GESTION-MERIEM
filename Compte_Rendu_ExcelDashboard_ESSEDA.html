1. Présentation du Skill

Le skill Excel Dashboard est un outil automatisé conçu pour transformer tout fichier Excel (.xlsx / .xls) en un dashboard HTML interactif, autonome et téléchargeable, aux couleurs institutionnelles de l'ENCG Settat.

1.1 Identité du Skill

AttributValeur
Nomexcel-dashboard
Couleur primaireVert ENCG #006633
Couleur secondaireOr ENCG #D4A017
Technologie graphiqueChart.js 4.x via CDN
Format de sortieFichier .html autonome tout-en-un

1.2 Déclencheurs d'Activation

Le skill s'active dès que l'utilisateur mentionne :

  • dashboard, tableau de bord, visualisation de données
  • visualiser mon Excel, graphiques depuis Excel, analyser mon fichier Excel
  • KPI depuis Excel, rapport visuel
  • montre-moi mes données, fais quelque chose de visuel avec ce fichier
  • Ou lorsqu'un fichier .xlsx / .xls est uploadé avec une demande visuelle
⚠️ Règle d'exclusion : Si le livrable attendu est un fichier Excel modifié, utiliser le skill xlsx à la place.

2. Principe Général et Livrables

2.1 Sortie Unique

Le skill produit un seul fichier : un .html entièrement autonome, sans aucune dépendance locale. Toutes les données sont embarquées en JSON directement dans le code JavaScript.

2.2 Types de Données Supportés

DomaineExemples de Données
Financier / ComptableBilan, compte de résultat, trésorerie, charges, produits
AcadémiqueNotes, mentions, modules, filières, résultats d'examens
Commercial / VentesCommandes, clients, produits, régions, remises
LogistiqueStocks, fournisseurs, délais, livraisons, entrepôts
GénériqueToute donnée tabulaire structurée non catégorisée

2.3 Stack Technique

  • Chart.js 4.x via CDN — pour tous les graphiques
  • Vanilla JavaScript — sans framework pour la portabilité maximale
  • CSS Grid / Flexbox — layout responsive et adaptatif
  • HTML5 — structure sémantique autonome

4. Sélection des Graphiques et KPI

4.1 Algorithme de Sélection

1. Colonne date présente       → LineChart (temporel) en priorité
2. Numérique vs Catégorielle   → BarChart horizontal ou vertical
3. ≤ 7 catégories + 1 métrique → DoughnutChart
4. ≥ 3 colonnes numériques     → RadarChart
5. 2 colonnes numériques       → ScatterChart
6. Toujours                    → 4 à 6 KPI Cards en haut de page
7. Toujours                    → Tableau paginé en bas

6. Gestion des Cas Particuliers

6.1 Fichier Multi-Feuilles

  • Créer un système de navigation par onglets dans le dashboard
  • Ou interroger l'utilisateur sur la feuille à analyser en priorité

6.2 Données Manquantes

  • Ignorer les colonnes avec > 50% de valeurs NaN
  • Afficher un avertissement visible si des colonnes ont été exclues
  • Ne jamais générer d'erreur : utiliser df.dropna() ou df.fillna(0)

6.3 Grands Volumes (> 1 000 lignes)

  • Agréger avant intégration JSON : groupby + sum / mean
  • Limiter le tableau paginé à 500 lignes maximum
  • Afficher le total dans un KPI "Total enregistrements"

6.4 Colonnes Dates Mal Formatées

for col in df.columns:
    try:
        df[col] = pd.to_datetime(df[col])
    except:
        pass

7. Checklist Qualité

Avant toute livraison, vérifier systématiquement :

  • Le fichier HTML s'ouvre sans erreur dans un navigateur moderne
  • Tous les graphiques se chargent correctement (aucun canvas vide)
  • Les KPI Cards affichent des valeurs cohérentes avec les données source
  • Le tableau est lisible et paginé (10 lignes / page)
  • Le design est cohérent : couleurs, typographie, espacements
  • Le fichier est entièrement autonome (CDN uniquement, zéro dépendance locale)
  • Le titre reflète fidèlement le contenu du fichier source
